What to Submit:

Demo submissions are a short (1-page) description of the software to be demoed. It should include contact information for whom to contact for access to the technology. This can be in the format of a brochure.

If accepted, please submit a 3-minute video enticing participants to come to your live demo. All accepted demos will be run simultaneously during demo sessions. All demo submissions, if accepted, will be published on-line only in the on-line conference proceedings.

Suggestion: Participants may want to use Jupyter to demonstrate their code and results. Examples can be found here, Sample Jupyter Notebooks.

Where to Submit:

All materials should be submitted directly to sbp-brims@andrew.cmu.edu

For accepted demos the description and the 3-minute video will be put on the conference website under the on-line proceedings.

What to do at the Conference:

If accepted, all demos will need to provide a 3-minute video. The demo presenter(s) are required to register for the conference.
